# MIKO_HANDOFF — Qwen3.5-2B 图像→Tag 任务 v0 复盘

> 记录 v0(`full_v0`,1 epoch 全参微调)的坑与解决方案,供下一轮实验直接接手。
> 数据/证据均来自 `saves/qwen3.5-2b/mikomiko/` 存档与 `data/mikomiko_tag/` 的实读统计(2026-07-07)。
> 所有结论标注了证据强度:**[实测]** = 全量/大样本统计;**[方向性]** = 仅 n=10 sanity,只表趋势。

## 1. 三个问题的结论

### Q1 — 验证指标是否合理?测试集是不是"未见+常见"?"鸽子"多大规模?

**核心坑:所有指标(F1 + BLEU + ROUGE)都只在 `sanity_10.jsonl` 的 n=10 上算的。** [实测]

- n=10 的 95% 置信区间半宽 ≈ **±10 个百分点**;两个不同 checkpoint(8000 vs 11530)microF1 都是 31.5、compF1 都是 8.3 —— **指标已无分辨力**,不能用来选 checkpoint / 判断收敛 / 做 A/B。
- 要 ±3 点需 ~86 张、±2 点需 ~193 张、±1 点需 ~770 张。**现成的 `*_mini`(各 200,种子=42,共 400)就能把 CI 压到 ±1.4 点**,却一直没被用来算验证分。
- **真正的全量测试集(12.3w unseen + 1945 strat)从没跑过**——这就是"鸽子":规模其实很大,只是没执行。

**指标含义与取舍:**

| 指标 | 含义 | 取舍 |
|---|---|---|
| micro/macro F1 | 无序 tag 集合的整体 / 每图平权命中 | ✅ 主看 |
| atomF1 | 单词 tag(词表仅 350,可学) | ✅ 反映基础识别,当前 46 已不低 |
| compF1 | 多词 tag **精确集合匹配** | ⚠️ 苛刻且被标签体系拖累,勿当模型质量单看 |
| tokF1 | 词级重叠 | ⚠️ 偏乐观 |
| BLEU-4 / ROUGE-2/L | seq2seq 序列指标 | ❌ 对无序 tag 列表**无意义/误导**,建议弃用;ROUGE-1 勉强可作词级召回参考 |

**测试集设计(合理但需澄清):** [实测]
- **只有 `test_unseen` 是真·未见人物**(整 post 留出,post 级零泄漏已全量验证)= OOD 泛化。
- `test_stratified` 与 train 在 post/人物层面 **1812/1814 重叠**(仅图像级零泄漏)= **同分布 + 稀有 tag 保底探针**,**不是"常见样本"**。它按"每图最稀有 tag 的 DF"分 5 band,但每图平均 11 个 tag,实例级仍是 very_common 占 ~55%,与随机留出几乎无差别。要看"常见 vs 稀有 tag 性能"应**按 tag 聚合**,而非按图。

### Q2 — LR 是否该增大?(4k-6k 后 loss 不降)

**结论:不该把加大 LR 当主手段。平台期是数据/任务天花板,不是优化受限。** [实测,证据链]

- **铁证**:eval_loss 在 **step 6000(lr 仍有 5.08e-6 = 峰值 51%)** 时已基本停降;后半程 lr 5e-6→0 只换来 0.002 改善。若是"LR 太低/被 cosine 压平",不该在 lr 还这么大时就平。
- train/eval gap 仅 ~0.08(1.05 vs 1.14)→ 不是过拟合,是两条线**一起撞到同一堵墙**,典型接近数据可学习上限。
- grad_norm 早早稳定在 ~8 且 max 单调收窄 → 优化平稳,没有"迈不动步"的迹象。
- 1e-5 本就在 VLM 全参 SFT 正常区间(1e-5~2e-5,vision tower 常更低/冻结)。
- tag 是主观多标签,存在不可消除的 aleatoric 熵,**1.14 的 CE 很可能已接近该标注分布的条件熵下限**。
- **推论:loss 平 ≠ tag 质量到顶**。该换成 tag 级 P/R/F1 来判断,而不是盯 loss。

### Q3 — 从"词本有限"出发解决复合 tag(compF1=8.3)

**结论:词表确实是硬闭集,但"受限解码/白名单"救不了 compF1——药开错了。** [词表 实测 / 预测侧 方向性]

- **闭集**:全量仅 **3145 个唯一 tag(350 原子 + 2795 复合)**,10 万行就饱和,**几乎无长尾**(DF≥5 仍保留全部 2795 复合)。复合是固定 board 体系,不是 Gemini 临场拼词。
- **但白名单无效**:模型预测的复合 **98.3% 本来就在词表内**(越界率 1.7%)。白名单/trie 受限解码把 compF1 从 8.3 只抬到 **8.4**。[方向性]
- **真正的失败模式 = 在合法词表里"选错 + 过量生成"**:每图吐 **5.8 个复合 vs 标签 3.8**,且偏爱高频通用复合(big tits / natural tits / big tits lesbian),合法但与图不符 → 是**精确率**问题。
- **只输出原子**把 microF1 从 31.5 抬到 **36.8**(P 30→48.6)→ 当前复合预测对总分是**净负贡献**。
- "只学原子 + 规则拼复合"上限低:即便给**金标原子**,词表约束重建 compF1 也仅 **18**(一组原子对应多个合法复合,规则无法判定用哪个)。

## 2. v0 的坑清单(踩过的 / 发现的)

| # | 坑 | 影响 | 证据 |
|---|---|---|---|
| P1 | **用 n=10 sanity 当验证指标** | 无法选 ckpt、无法判断收敛,误以为"事不理想" | 两 ckpt 指标完全相同;CI ±10 [实测] |
| P2 | **全量 test(12.3w+1945)从没跑** | 手上没有可信的泛化数字 | 只有 predict_sanity/ 存在 |
| P3 | **BLEU/ROUGE 被当指标** | 对无序 tag 列表误导 | all_results.json 里的 bleu/rouge |
| P4 | **误判 LR 是瓶颈** | 差点浪费一轮去拉 LR | eval 在 lr=51% 峰值时已平 [实测] |
| P5 | **复合 tag 想靠受限解码救** | 方向错,越界率只有 1.7% | 白名单 8.3→8.4 [方向性] |
| P6 | **模型过量生成复合** | 拖低精确率,净负贡献 | 每图 5.8 vs 3.8 [方向性] |
| P7 | **标签噪声高**:同义词未归一(cock/dick、black/ebony/african)、**26.6% 复合可被同图单词完全拆出**(冗余)、df=1 一次性复合 | compF1 天花板可能只有 20~30%,microF1 上限约 50~60% | train/test 抽样 [实测] |
| P8 | **strat 混入驼峰拼接标签**(`EbonyMILF`/`BlackLesbian`),train=0 例、strat=24 处 | train 从没这种写法 → 模型必不匹配;被 `norm()` 当原子;n=10 里一条样本含 ~6 个被严重拖低 | 本次核实 [实测] |

## 3. 解决方案与下一步(按性价比排序)

### 第一优先 —— 零/低训练成本,先把"看不清"解决(对应 P1-P4)

1. **[必做,零训练] 重算可信验证指标。**
   - 立刻:用 `full_v0` 现有 ckpt,在 **400 条 mini**(unseen+strat 各 200,种子已固定)上算分 → CI ±1.4,能选 ckpt。
   - 终评:vLLM 批量推理跑**全量 test_unseen + test_stratified**,离线打分。
   - 报告口径:①按 **unseen vs stratified 分组**;②**micro/macro 的 P 和 R 分开报**(看模型是"话痨"还是"保守");③新增**按 tag 的 train-DF band 聚合的每-tag 召回**(这才是"常见 vs 稀有 tag"的正确切法);④compF1 同时报 exact 与 token-subset 两版;⑤**弃用 BLEU/ROUGE-2/L**。
   - 评测脚本改动:复用 `instruction_follow_check_mikomiko.sh` 里的 `norm/tagset/prf`,去掉 BLEU/ROUGE,history 加 `n` 与 `CI` 列,防止再拿 n=10 比较。

2. **[必做,零训练] 确认该不该继续在 loss 上较劲。** 用上面的 tag 级 P/R 判断:loss 平了但 tag 质量大概率仍有空间(尤其精确率)。

### 第二优先 —— 直接提总分的低成本手段(对应 P5/P6)

3. **[低成本] 治过量生成**:后处理**限制复合输出数量 / 提高复合解码阈值**,或训练侧对复合数量加正则,把 5.8 压回 ~3.8。若业务更看重整体 microF1,"少输出/不输出低置信复合"可直接 31.5→36.8。
4. **[保险丝,别当主力] 生成后白名单过滤**越界 tag(10 行代码):保证合法性,但对 compF1 增益≈0,不要指望它提分。

### 第三优先 —— 治本(对应 P5 选错 + P7 噪声)

5. **[推荐主攻,低-中成本] 把 2795 个复合闭集显式喂进 prompt / 候选清单**,让模型从"生成"变"选择"。这是把"词本有限"这一事实真正用起来的最对症手段。
6. **[冲高 compF1] multi-label 分类头**:把 3145 个 tag 当 multi-label 分类,每个 tag 出概率按阈值输出。天然治过量生成,阈值可直接对 compF1/microF1 调优。偏离生成范式,成本中等。
7. **[数据侧,低成本] 标签去噪**:建同义词归一表(cock/dick、black/ebony/african…)、合并可完全拆分的冗余复合、过滤 df=1 一次性复合、**统一驼峰拼接标签写法(P8)**。既抬评测天花板,也降监督歧义。

### LR / 训练超参(对应 P4)——不是重点,但给出证伪实验

8. **[可选证伪] LR 短跑对比**:peak **2e-5**(勿超 3e-5),warmup 0.03,cosine **不衰减到 0**(留 ~10% 峰值),只跑到 step 4000-5000(~1.5h)对比 eval。预测:与 baseline 基本重合或略差 → 坐实天花板。
9. **[可选] freeze_vision_tower 对比**:判断视觉塔全参是被主观标签噪声带偏还是有用,顺带省显存。
10. **[可选] 多 epoch(2-3)**:gap 小,预期边际收益,放最后。

---

## 4. 推荐执行顺序(TL;DR)

1. 跑 **400 mini 重评** + 全量 test 评测,拿到可信数字(分组 + 分 band + P/R 分开),弃 BLEU/ROUGE。**先看清,再动手。**
2. 用 tag 级 P/R 确认瓶颈:大概率是**精确率(过量+选错复合)**,而非 loss。
3. **治过量生成**(阈值/正则)拿一次快速提分;白名单仅作保险丝。
4. 主攻**闭集喂 prompt**(治本、低成本);要冲 compF1 再上 **multi-label 分类**。
5. **标签去噪**(同义归一 / 去冗余复合 / 统一驼峰写法)抬天花板。
6. LR/epoch 仅作证伪实验,不是主线。

## 附:证据强度与残留风险

- **[实测,可靠]**:训练量=1 epoch、有效 batch 96、平台归因、词表 3145 闭集、测试集泄漏核查、标签噪声率、驼峰标签 P8。
- **[方向性,需放大样本复核]**:所有"预测侧"数字(越界率、5.8 vs 3.8、白名单 8.3→8.4、只输出原子 36.8、重建上限 18)均来自 **n=10** predict_sanity,**先做第 1 步重评后再据此定策**。
- token 数(~3-4 亿)由 total_flos 反推;`num_input_tokens_seen` 未开启(=0)。
- "1.14 ≈ 数据熵下限"是强推断,非直接测量,严格证实需 LR 证伪实验 + tag 级指标。
